# Regole Componenti React

- Componenti funzionali con hooks (no classi)
- Ogni componente ha il proprio file CSS con lo stesso nome
- Props destructurate nel parametro della funzione
- Stato locale con `useState`, effetti con `useEffect`
- Gestire sempre stati di loading e errore nelle chiamate API
- Nomi componenti in PascalCase, file in PascalCase
- Testi UI in italiano
